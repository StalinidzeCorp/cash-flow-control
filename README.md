# ДДС — Движение денежных средств

Веб-приложение для учёта движения денежных средств. Позволяет создавать, редактировать, удалять и просматривать записи о доходах и расходах с фильтрацией и управлением справочниками.

## Технологии

- Python 3.14, Django 6.0.6
- SQLite
- Bootstrap 5 (CDN)
- Django Templates (DTL)

## Установка и запуск

### 1. Установить зависимости

Проект использует `uv` для управления зависимостями.

```bash
git clone <repo-url>
cd cash-flow-control
uv sync
```

### 2. Настроить базу данных

```bash
uv run python manage.py migrate
```

Миграции создадут все необходимые таблицы и заполнят справочники начальными данными (статусы: Бизнес, Личное, Налог; типы: Пополнение, Списание).

### 3. Запустить веб-сервис

```bash
uv run python manage.py runserver
```

Приложение будет доступно по адресу http://localhost:8000/

### Создание учётной записи

При первом входе зарегистрируйтесь через `/accounts/register/` или создайте суперпользователя для доступа в админ-панель:

```bash
uv run python manage.py createsuperuser
```

**Предустановленный администратор** (логин/пароль):
- **Логин:** `admin`
- **Пароль:** `admin`

## Функциональность

### Записи ДДС
- **Создание** записи: дата, статус, тип, категория, подкатегория, сумма, комментарий
- **Редактирование** любой записи
- **Удаление** записи
- **Фильтрация** по дате (период), статусу, типу, категории, подкатегории

### Справочники
- Управление статусами (Бизнес, Личное, Налог)
- Управление типами (Пополнение, Списание)
- Управление категориями (привязаны к типу)
- Управление подкатегориями (привязаны к категории)

### Логические зависимости
- Категория должна относиться к выбранному типу
- Подкатегория должна относиться к выбранной категории
- Защита от удаления справочников, используемых в записях

### Валидация
- Клиентская (JavaScript): обязательные поля, корректность суммы
- Серверная (Django Forms): проверка связей тип→категория→подкатегория
